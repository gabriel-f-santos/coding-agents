# Pinned stack — Frontend: Vite + React (TS) → Cloudflare Workers

Open when the user picks the **frontend-app = React SPA on Cloudflare Workers** stack.
A React + TypeScript SPA served by a Worker (static assets) with an optional same-Worker API,
living as one package inside the monorepo. Plain React (no Next/Router framework).

> Versions below were current 2026-06-14. **Before emitting, re-verify** with
> `npm view <pkg> version` — pin caret (`^`) to the current major. (These are a major jump over
> commonly-cached Vite 6 / Vitest 2 / ESLint 9.)

## Decisions (the defaults to scaffold)
- Use **Vite + `@cloudflare/vite-plugin`** — NOT Pages, NOT a bare Worker. The plugin runs the
  Worker in the real `workerd` runtime during `vite dev`, so dev == prod. Pages is legacy for new SPAs.
- Quick path: `npm create cloudflare@latest -- <app> --framework=react` (wires plugin, wrangler,
  tsconfig split, ESLint flat config). Full control: emit the files below directly.
- Static assets via the **`assets`** block in `wrangler.jsonc`; SPA fallback =
  `"not_found_handling": "single-page-application"`. Co-located API → `run_worker_first: ["/api/*"]`.
- Commands: `vite dev` (port **5173**), `vite build`, `wrangler deploy`, `wrangler types` (→ `cf-typegen`).
- Lint: ESLint **flat config** (`eslint.config.js`), `typescript-eslint` + react-hooks + react-refresh.
  Prettier kept **separate** (formatting out of ESLint).
- Test: **`@cloudflare/vitest-pool-workers`** for Worker/API tests (runs in workerd); plain
  `vitest` + jsdom for pure component tests.

## Folder layout (SPA + same-Worker API)
```
apps/web/
├── index.html                # SPA entry (Vite root)
├── public/                   # copied as-is into the asset bundle
├── src/                      # CLIENT code (browser bundle): main.tsx, App.tsx, assets/
├── worker/index.ts           # Worker entry: API + serves assets (the `main`)
├── test/worker.spec.ts       # + test/tsconfig.json
├── vite.config.ts
├── wrangler.jsonc            # at the PACKAGE root, next to package.json
├── tsconfig.json             # solution file → references app/node/worker configs
├── tsconfig.{app,node,worker}.json
├── eslint.config.js
├── worker-configuration.d.ts # generated by `wrangler types`
└── package.json
```
For a **pure SPA with no API**, drop `worker/` + `main` and keep only `assets.not_found_handling`.

## Minimal files to emit

`wrangler.jsonc`
```jsonc
{
  "$schema": "./node_modules/wrangler/config-schema.json",
  "name": "web",
  "compatibility_date": "<scaffold-date>",
  "compatibility_flags": ["nodejs_compat"],   // only if the Worker imports Node built-ins
  "main": "./worker/index.ts",
  "assets": {
    "not_found_handling": "single-page-application",
    "run_worker_first": ["/api/*"]
  },
  "observability": { "enabled": true }
}
```

Hello-world `worker/index.ts` + `src/App.tsx` + `test/worker.spec.ts`:
```ts
// worker/index.ts
export default {
  fetch(request) {
    const url = new URL(request.url);
    if (url.pathname.startsWith("/api/")) return Response.json({ name: "Cloudflare" });
    return new Response(null, { status: 404 });
  },
} satisfies ExportedHandler<Env>;            // Env comes from worker-configuration.d.ts
```
```tsx
// src/App.tsx
import { useState } from "react";
export default function App() {
  const [name] = useState("Cloudflare");
  return <h1>Hello {name}!</h1>;
}
```
```ts
// test/worker.spec.ts
import { env, createExecutionContext, waitOnExecutionContext } from "cloudflare:test";
import { describe, it, expect } from "vitest";
import worker from "../worker";

describe("hello worker", () => {
  it("returns JSON on /api/", async () => {
    const ctx = createExecutionContext();
    const res = await worker.fetch(new Request("http://x/api/"), env, ctx);
    await waitOnExecutionContext(ctx);
    expect(await res.json()).toEqual({ name: "Cloudflare" });
  });
});
```

## Gotchas (call these out)
- **Node built-ins** (`fs`/`path`/node `crypto`) need `compatibility_flags: ["nodejs_compat"]` + a recent
  `compatibility_date` — and only a subset exists. Omit the flag if the Worker doesn't import Node.
- **Secrets, not `.env`**: client `import.meta.env` is the PUBLIC browser bundle. Worker secrets go via
  `wrangler secret put <NAME>` (prod) and a **gitignored `.dev.vars`** (local). Never put secrets in
  `wrangler.jsonc` `vars`. Add `.dev.vars` to `.gitignore`.
- **SPA fallback swallows `/api/*`** without (correctly-scoped) `run_worker_first` → returns `index.html`.
- **Don't set `assets.directory` manually** when the vite-plugin is in use — the plugin manages it.
- **Regenerate types after binding changes**: stale `worker-configuration.d.ts` → `Env` mismatches; rerun
  `wrangler types` (`cf-typegen`).
- **Bundle size**: Worker script limit ~3 MB gz (paid)/1 MB (free) — keep big libs in the client bundle.
